Алгоритм работы:
1. Указать xsd-схему описываемого элемента/комплексного типа.
2. Указать элемент/комплексный тип из имеющихся в xsd-схеме.
3. Сформировать csv-файл:
	Имя файла : Формат <название типа/комплексного типа>
	Колонки:
		Номер - начинается с "1". Инкриментируется для каждого элемента. Для вложенных элементов также углубляется (1.1, 1.2.1). Для choice имеет вид: 3.2(choice1), 3.2(choice2).. - см. пример в "Формат createLoanRequest.csv".
		Название поля - название элемента
		Тип поля - секция*, элемент, аттрибут (секция* - содержит хотя бы один элемент. Если это трудно, то вместо "секции" можно писать "элемент")
		Тип - тип элемента
		Обяз. - обязательность. Значения: 0..1, 0..n, 1..1, 1..n.
		Комментарий - комментарий из annotation.documentation данного элемента или атрибута
	
	Пример формата "Формат createLoanRequest.csv"
4. Конец.

Пример:
1. LoanService.xsd
2. createLoanRequest - элемент
3. Формат createLoanRequest.csv
